title: "SaveToWin mainnet program and token addresses"
description: "All on-chain addresses used by SaveToWin — program IDs, token mints, Voltr vault, Kamino reserve, and PT-ONyc market — exported as typed SDK constants."
---

All addresses used across the SaveToWin protocol are exported from `@savetowin/sdk` as typed `PublicKey` constants. Every package in the monorepo — the CLI, strategy bot, and Next.js frontend — imports from this single source of truth. Changing a deployed address means changing one file.

Import any constant directly:

<CodeGroup>

```typescript import-example.ts
import {
  PRIZE_VAULT_PROGRAM_ID,
  USDC_MINT,
  VOLTR_VAULT,
  VOLTR_LP_MINT,
  DEFAULT_EPOCH_DURATION_SEC,
  MAX_STREAK,
  DEFAULT_ROLLOVER_BPS,
  DEFAULT_YIELD_SPLIT_BPS,
} from "@savetowin/sdk";
```

</CodeGroup>

## Program defaults

These numeric constants are the default values used by the SaveToWin vault configuration.

| Constant | Value | Description |
|---|---|---|
| `DEFAULT_EPOCH_DURATION_SEC` | `86400` | Epoch length in seconds (1 day) |
| `DEFAULT_YIELD_SPLIT_BPS` | `2000` | Percentage of yield sent to the prize pool (20%) |
| `DEFAULT_ROLLOVER_BPS` | `1000` | Percentage of the prize pool retained on rollover (10%) |
| `MAX_STREAK` | `5` | Maximum streak count; caps the streak multiplier at 2^5 = 32× |
| `BPS_DENOMINATOR` | `10000` | Basis point denominator |
| `MAX_OBSERVATIONS` | `64` | Maximum TWAB observation slots per user account |

<Note>
  `DEFAULT_YIELD_SPLIT_BPS = 2000` means 20% of harvested yield goes into the prize pool and 80% is retained by the vault's depositors as additional USDC value.
</Note>
