## Novelty — auto-rolling on-chain market discovery

The strategy bot has **no hardcoded PT market address**. Every epoch it:

1. Calls `listAllMarkets(exponentProgram)` → returns all 94 `MarketTwo` accounts
2. Filters for ONyc-backed SY mint whitelist (maturity + liquidity + APY bounds)
3. Ranks by score = maturity-proximity + APY + log10(depth)
4. Picks the best live candidate
5. If held PT is matured (next epoch), redeems first, then deploys into the new market

Verified live on mainnet:

```
[scan]  94 MarketTwo accounts fetched in 1380ms
[cat]   3 ONyc-backed markets: 2 matured, 1 live
[pick]  ✅ 8QJRc12BDXHR... — 29.6d, 13.87% APY, $3.69B SY depth
[+30d]  no live → auto-fallback to 100% Kamino ✅
```

Code location: `strategy-bot/src/adapters-sdk/exponent.ts` → `findBestOnycMarket()`.

## How to run the full demo

```bash
git clone https://github.com/jokilvered/savetowin.git
cd savetowin

# 1. Surfpool E2E (requires surfpool running locally)
cd cli && npx ts-node ../scripts/demo-triple-flywheel.ts

# 2. Mainnet auto-market discovery (public RPC, no keys needed)
cd strategy-bot && npx ts-node --transpile-only ../scripts/test-market-discovery.ts

# 3. Mainnet Jupiter quote roundtrip (public RPC, no keys needed)
cd strategy-bot && npx ts-node --transpile-only ../scripts/test-jupiter-mainnet.ts

# 4. Decode real StreakUpdated events from mainnet
cd strategy-bot && npx ts-node --transpile-only ../scripts/capture-mainnet-events.ts
```
